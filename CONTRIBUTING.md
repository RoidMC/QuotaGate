# Contributing to QuotaGate

Thank you for your interest in contributing to QuotaGate! This document outlines the guidelines for contributing to the project.

## Code of Conduct

All contributors are expected to adhere to the project's Code of Conduct.

## License

## Contributor License Agreement

Before submitting any contributions, you must sign the [Contributor License Agreement](CLA.md). This agreement grants the project owner the right to sublicense contributions under commercial terms while maintaining open-source availability.

To sign the CLA, include the following in your contribution submission (pull request, merge request, or other submission mechanism):

> I have read the CLA Document and I hereby sign the CLA

For corporate contributions, include the entity name:

> I have read the CLA Document and I hereby sign the CLA on behalf of [Entity Name]

## How to Contribute

### Reporting Issues

- Use the project's issue tracker to report bugs or request features
- Include detailed information about the issue, including steps to reproduce
- Label issues appropriately

### Submitting Contributions

1. Fork the repository and create a feature branch
2. Make your changes
3. Write tests for your changes
4. Ensure all tests pass
5. Sign the CLA in your contribution submission
6. Submit the pull request or merge request

### Code Style

- Follow Go conventions (run `go fmt` before submitting)
- Write clear, concise commit messages
- Include comments for complex logic
- Add tests for new functionality

### Testing

- Run the full test suite with `go test ./...`
- Ensure all tests pass before submitting
- Add new tests for any new functionality

## Dual Licensing

QuotaGate is distributed under MPL 2.0 for open-source usage. Commercial licensing options are available for proprietary use cases. Contributions submitted under this CLA enable the project owner to offer commercial licenses while maintaining the open-source version.

## Questions

If you have any questions about contributing, please open an issue or contact the maintainers.
